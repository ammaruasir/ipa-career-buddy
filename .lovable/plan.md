## خطة إكمال الميزات الناقصة — للوصول لمستوى HireVue / Modern Hire

بناءً على التحليل السابق، النظام يغطي ~65% من ميزات المنافسين. هذه خطة لإكمال الـ 35% المتبقية، مرتبة حسب الأولوية:

---

### المرحلة 1: نظام تتبع مراحل التوظيف (Hiring Pipeline)

**الهدف**: إضافة Kanban board في لوحة HR لتتبع المرشحين عبر مراحل التوظيف

- إضافة عمود `pipeline_stage` لجدول `job_applications` (قيم: applied → screening → interviewing → offered → hired → rejected)
- إنشاء صفحة `/dashboard/hr/pipeline` بأعمدة Kanban مع سحب وإفلات بين المراحل
- تحديث حالة الطلب تلقائياً عند نقل البطاقة
- إضافة فلترة حسب الوظيفة والقسم

**الملفات**: migration جديد + صفحة `HiringPipeline.tsx` + تعديل `App.tsx` + تعديل `HRDashboard.tsx` (رابط سريع)

---

### المرحلة 2: تصدير تقارير PDF احترافية

**الهدف**: تمكين HR من تصدير تقرير تقييم المرشح كملف PDF

- إنشاء Edge Function `generate-report` تستخدم AI لتنسيق التقرير
- التقرير يشمل: بيانات المرشح، درجات التقييم، نقاط القوة/التحسين، توصية AI، تحليل DISC
- إضافة زر "تصدير PDF" في صفحة `CandidateDetail.tsx`
- استخدام مكتبة jsPDF أو إرجاع HTML محول لـ PDF من الـ Edge Function

**الملفات**: Edge Function جديد + تعديل `CandidateDetail.tsx`

---

### المرحلة 3: مقارنة المرشحين جنباً لجنب

**الهدف**: تمكين HR من مقارنة عدة مرشحين على نفس الوظيفة

- إنشاء صفحة `/dashboard/hr/compare` 
- اختيار 2-4 مرشحين من نفس الوظيفة
- عرض جدول مقارنة: الدرجة الكلية، المهارات التقنية، التواصل، الشخصية، نسبة تطابق السيرة الذاتية
- Radar chart لمقارنة بصرية

**الملفات**: صفحة `CandidateCompare.tsx` جديدة + تعديل `App.tsx` + تعديل `HRDashboard.tsx`

---

### المرحلة 4: نظام الإشعارات

**الهدف**: تنبيهات فورية للمرشحين و HR

- إنشاء جدول `notifications` (user_id, type, title, message, read, created_at)
- إضافة trigger في قاعدة البيانات عند:
  - تقديم طلب جديد → إشعار لـ HR
  - تحديث حالة الطلب → إشعار للمرشح
  - اكتمال تقييم المقابلة → إشعار للمرشح و HR
- إنشاء مكون `NotificationBell` في الـ header مع عداد الإشعارات غير المقروءة
- استخدام Supabase Realtime لتحديث فوري

**الملفات**: migration + جدول `notifications` + مكون `NotificationBell.tsx` + تعديل headers في جميع الـ dashboards

---

### المرحلة 5: قسم الإرشاد المهني (Career Guidance)

**الهدف**: تحليل AI للمسار المهني المناسب للمرشح

- إنشاء Edge Function `career-guidance` تحلل مهارات السيرة الذاتية وتقترح:
  - مسارات مهنية مناسبة
  - مهارات مطلوبة للتطوير
  - وظائف مطابقة من الوظائف المتاحة
- إنشاء صفحة `/career-guidance` في لوحة المرشح
- عرض النتائج بتصميم بطاقات مع رسوم بيانية

**الملفات**: Edge Function جديد + صفحة `CareerGuidance.tsx` + تعديل `App.tsx` + تعديل `CandidateDashboard.tsx`
