

## تحليل الوضع الحالي

### هل المنصة تدعم الهاتف؟
**نعم تقنياً** — التطبيق مبني بـ React و Tailwind CSS وهو responsive. المقابلة **النصية** تعمل بشكل كامل على الهاتف. المقابلة **الصوتية** تعمل أيضاً لأنها تستخدم `navigator.mediaDevices.getUserMedia({ audio: true })` المدعوم في متصفحات الهاتف. المقابلة **بالفيديو** تعمل نظرياً لكنها تحتوي على مكون 3D Avatar قد يكون ثقيلاً على بعض الأجهزة.

### المشكلة الفعلية
عند التقديم على وظيفة من صفحة `/jobs`، يتم توجيه المرشح **تلقائياً للمقابلة النصية فقط** (السطر 94 في `JobVacancies.tsx`):
```
navigate(`/interview/text?job=...&vacancy_id=...`)
```

لا توجد طريقة لاختيار نوع المقابلة (صوتية أو فيديو) عند التقديم.

## الخطة: إضافة اختيار نوع المقابلة عند التقديم

### 1. إضافة Dialog لاختيار النوع في صفحة الوظائف
- عند الضغط على "تقديم"، يظهر Dialog يتيح للمرشح اختيار: **نصية** أو **صوتية** أو **فيديو**
- كل خيار يعرض أيقونة ووصف مختصر (مثلاً: "الصوتية - سجّل إجاباتك صوتياً")
- بعد الاختيار يتم التوجيه إلى المسار المناسب: `/interview/text` أو `/interview/voice` أو `/interview/video`

### 2. تحسين تجربة الهاتف
- إضافة تنبيه في خيار الفيديو: "يُفضل استخدام جهاز كمبيوتر لأفضل تجربة"
- التأكد من أن الـ Dialog متجاوب ويعمل على الشاشات الصغيرة

### الملفات المتأثرة
- `src/pages/JobVacancies.tsx` — تعديل `applyToJob` لعرض Dialog بدل التوجيه المباشر
