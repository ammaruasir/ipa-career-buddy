

## تحليل المشكلة

أزرار الإجراءات في قسم "ملاحظات الموارد البشرية" (قبول، رفض، إعادة المقابلة، انتظار) لا تختفي أبداً بعد الضغط عليها لأن الكود الحالي (سطر 472-477) يعرضها دائماً بدون أي شرط. بينما أزرار "إطلاق النتائج / رفض" في أعلى الصفحة تعمل بشكل صحيح لأنها مشروطة بـ `reviewStatus === "pending_review"`.

## التعديل المطلوب — `src/pages/CandidateDetail.tsx`

1. **تتبع آخر إجراء مُتخذ**: فحص `notes` للبحث عن آخر إجراء حاسم (accepted/rejected) — إذا وُجد، إخفاء الأزرار وعرض حالة القرار بدلاً منها
2. **عرض شريط الحالة**: عند وجود قرار سابق، عرض Badge بالإجراء المتخذ مع زر "تغيير القرار" لإعادة إظهار الأزرار إذا أراد المسؤول تعديل قراره
3. **المنطق**: 
   ```text
   const lastDecision = notes.find(n => ["accepted", "rejected"].includes(n.action));
   إذا وُجد قرار → عرض Badge + زر تغيير
   إذا لم يوجد → عرض الأزرار كالعادة
   ```
