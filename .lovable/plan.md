## المشكلة
حالياً صفحة "تحليل ودردشة السيرة" تعرض اقتراحات الذكاء الاصطناعي (`rewrites`) كنصوص للقراءة فقط، بدون أي زر لاعتمادها، ولا حقل لتعديلها يدوياً، ولا إمكانية لتصدير نسخة محسّنة بصيغة Word أو PDF.

## الخطة

### 1. قاعدة البيانات (Migration)
إضافة جدول `cv_revisions` لحفظ النسخة المعتمدة من السيرة:
- `id`, `user_id`, `cv_document_id`
- `content` (JSONB) — بنية الأقسام بعد التعديل: `{ summary, experience[], education[], skills[], achievements[], contact }`
- `accepted_rewrites` (JSONB) — قائمة بالـ rewrites المعتمدة
- `created_at`, `updated_at`
- RLS: المستخدم يدير سجلاته فقط
- Trigger للتحديث التلقائي لـ `updated_at`

### 2. واجهة CVReview.tsx — منطقة "التحسينات"
لكل rewrite يظهر:
- النص الأصلي (للمقارنة)
- النص المقترح من AI (قابل للتعديل عبر `Textarea`)
- زرّان: **"اعتماد"** (يحفظ النسخة المعدّلة في `cv_revisions.accepted_rewrites`) و **"تجاهل"**
- شارة حالة (معتمد / مُعدّل / مرفوض)

كذلك في الدردشة (`CVChatPanel`): عند توليد AI لمقترح إعادة كتابة، يظهر زر **"إضافة إلى التحسينات المعتمدة"** أسفل الرسالة.

### 3. صفحة "السيرة النهائية" + التصدير
قسم جديد أعلى الصفحة بعنوان "سيرتي المحسّنة" يعرض:
- معاينة نصية للنسخة بعد دمج جميع التحسينات المعتمدة
- زر **"تصدير Word (.docx)"**
- زر **"تصدير PDF"**

### 4. توليد الملفات (Frontend)
- **DOCX**: استخدام مكتبة `docx` (npm) لبناء المستند بأقسام عربية RTL، خط Arial، عناوين بارزة، نقاط لكل خبرة.
- **PDF**: استخدام `jspdf` + `jspdf-autotable` مع خط عربي (Noto Naskh Arabic مضمّن كـ base64) لضمان عرض صحيح RTL.
- التصدير يتم في المتصفح مباشرة بدون edge function (لا حاجة لرفع الملف).

### 5. التفاصيل التقنية
- `extraction` (jsonb) الموجود في `cv_documents` يُستخدم كأساس للنص، ثم تُطبّق `accepted_rewrites` فوقه عبر مطابقة `original → improved`.
- إن لم تكن النسخة المعتمدة موجودة، يستخدم زر التصدير `extraction` كما هو.
- المكتبات المضافة: `docx`, `jspdf`, `file-saver`.

### 6. ملفات سيتم تعديلها/إنشاؤها
- `supabase/migrations/...` (جديد) — جدول `cv_revisions`
- `src/pages/CVReview.tsx` — أزرار الاعتماد، حقول التعديل، قسم التصدير
- `src/components/cv-builder/CVChatPanel.tsx` — زر "إضافة للتحسينات"
- `src/lib/cv-export.ts` (جديد) — دوال `exportToDocx` و `exportToPdf`
- `src/hooks/useCvRevision.ts` (جديد) — تحميل/حفظ النسخة المعتمدة

### 7. لماذا لم تكن موجودة سابقاً؟
الإصدار الحالي ركّز على **التحليل والعرض** فقط، ولم يُبنَ بعد منطق "اعتماد + تصدير". هذه الخطة تُكمل الميزة.
